## Decision Log

| Decision | Rationale |
| --- | --- |
| Host-based routing is the target architecture. | `b4r7.dev` should serve hub/future app index, `diffgazer.b4r7.dev` should serve product landing, `docs.b4r7.dev` should serve docs, and `r.b4r7.dev` should serve the shadcn registry. |
| `r.b4r7.dev` remains the canonical registry origin. | It matches `libs/registry/src/constants.ts:6`, docs metadata, Docker args, and compose defaults. It isolates registry CORS/cache/content-type policy from the docs runtime. |
| Docs should be treated as a TanStack Start/Nitro Node runtime with prerendering, not static-only. | Docker runs `.output/server/index.mjs`; docs use server functions/search. Static `.output/public` preview is not equivalent production coverage. |
| The local `diffgazer` API may keep the CSP/token-in-SPA model for now. | Current code binds packaged server to loopback, uses CSP nonce, Host/CORS/token guards, and rejects unsafe non-localhost origins. Cookie migration is not mandatory without new XSS evidence. |
| Unpublished packages and NXDOMAIN hosts are not source security bugs. | They are handoff gates. Docs must not expose dead commands as ready before deploy/publish is complete. |
| Required `@diffgazer/keys` peer for `@diffgazer/ui` package consumers is a policy choice. | It is documented and smoked today. If low-friction simple-component installs matter more, make it optional and add targeted package smokes. |
