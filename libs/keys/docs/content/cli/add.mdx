---
title: dgadd add keys
description: Install @diffgazer/keys hooks into your project.
---

## Copy Standalone Hooks

```bash
pnpm exec dgadd add keys/navigation
pnpm exec dgadd add keys/focus-trap keys/focus-restore keys/scroll-lock
```

Before publication, install a locally packed `@diffgazer/add` tarball into the target app and run `pnpm exec dgadd`. Use `npx @diffgazer/add add ...` only after `npm view @diffgazer/add version` succeeds.

Standalone hooks are copied into your project as source files and have no runtime dependency on `@diffgazer/keys`.

## Use The Package

Provider-backed hooks (`useKey`, `useScope`, `useScopedNavigation`, `useActionRowNavigation`, `useFocusZone`) are intentionally **package-only**. They depend on `KeyboardProvider` runtime context, so copying isolated source files would not give a working hook -- you would still have to bring `KeyboardProvider` along, which means depending on the package anyway. Install via npm:

```bash
npm install @diffgazer/keys
```

This package install is publish-gated until `npm view @diffgazer/keys version` succeeds.

```tsx
import { KeyboardProvider, useKey, useScope, useScopedNavigation, useActionRowNavigation, useFocusZone } from "@diffgazer/keys"
```

The CLI's `keys/*` namespace only exposes hooks that work standalone without a provider:

| `dgadd add keys/...` | Available |
|----------------------|-----------|
| `keys/navigation` | yes -- standalone |
| `keys/focus-trap` | yes -- standalone |
| `keys/focus-restore` | yes -- standalone |
| `keys/scroll-lock` | yes -- standalone |
| `keys/key`, `keys/scope`, `keys/scoped-navigation`, `keys/action-row-navigation`, `keys/focus-zone` | no -- package-only |

UI components that depend on provider hooks can be installed in package integration mode:

```bash
pnpm exec dgadd add ui/menu --integration keys
```
