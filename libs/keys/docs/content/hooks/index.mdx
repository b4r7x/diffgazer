title: Hooks
description: Per-hook API reference for @diffgazer/keys.
---

Standalone and provider-backed keyboard hooks. Standalone hooks work without a provider; the rest require `KeyboardProvider`.

- [useNavigation](/keys/hooks/use-navigation) — DOM-driven list and tab navigation
- [useFocusTrap](/keys/hooks/use-focus-trap) — trap Tab and Shift+Tab within a container
- [useFocusRestore](/keys/hooks/use-focus-restore) — capture and restore focus around temporary UI
- [useScrollLock](/keys/hooks/use-scroll-lock) — ref-counted scroll prevention
- [useKey](/keys/hooks/use-key) — bind keyboard shortcuts (requires provider)
- [useScope](/keys/hooks/use-scope) — push a named scope onto the stack (requires provider)
- [useScopedNavigation](/keys/hooks/use-scoped-navigation) — scope-aware list navigation (requires provider)
- [useActionRowNavigation](/keys/hooks/use-action-row-navigation) — row and action-cell navigation (requires provider)
- [useFocusZone](/keys/hooks/use-focus-zone) — multi-zone navigation with transitions (requires provider)
