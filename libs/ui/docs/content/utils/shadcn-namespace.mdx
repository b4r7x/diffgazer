---
title: shadcn Namespace
description: Configure shadcn registry aliases for @diffgazer/ui and @diffgazer/keys.
---

The shadcn CLI can install registry items by direct URL or by namespace after `components.json` maps each namespace to a registry URL.

Public Diffgazer registry hosts are future, non-blocking release work until deployment and endpoint smoke tests verify them. Until those hosts are live, use a local preview server or an environment-specific URL that serves the generated `public/r/{name}.json` files.

Public direct-URL item JSON carries absolute URLs for transitive Diffgazer registry dependencies. Configure both namespaces when you want shorthand for top-level UI and keys installs:

```json
{
  "registries": {
    "@ui": "https://<ui-registry-host>/r/{name}.json",
    "@diffgazer-keys": "https://<keys-registry-host>/r/{name}.json"
  }
}
```

For local validation, either rebuild artifacts with `REGISTRY_ORIGIN` set to the localhost registry host or use a local server that rewrites `/r/ui/*` and `/r/keys/*` dependency URLs to localhost. Do not publish concrete production hosts until deployment and endpoint smoke tests are complete.

Then install namespaced items:

```bash
npx shadcn@latest add "@ui/accordion"
npx shadcn@latest add "@ui/dialog" "@ui/popover"
npx shadcn@latest add "@ui/compose-refs"
```

When a UI source item depends on keys behavior, the published public JSON resolves that dependency through the corresponding keys registry URL.

## CSS Wiring

Namespaced shadcn installs copy component source and dependencies, but they do not make a CSS-dependent component complete by themselves. Import the canonical aggregated component CSS in the target app CSS entrypoint:

```css
@import "tailwindcss";
@import "./styles/styles.css";
```

That `styles.css` entry must be materialized from the shared theme seed plus registry component CSS such as dialog backdrop and body-lock rules. If you install directly from a served shadcn registry, verify the app receives the same aggregated CSS entry before treating components such as Dialog, CommandPalette, Select, Popover, or Tooltip as fully wired.

## Validation

For a handoff-ready release, verify a clean shadcn app can run:

```bash
npx shadcn@latest add "@ui/accordion" "@ui/dialog" "@ui/popover"
```

Then run the app type-check/build. Do not publish shadcn instructions for a registry host until those URLs serve the generated JSON files.
