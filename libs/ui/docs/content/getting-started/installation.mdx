`@diffgazer/ui` components install as source files. `dgadd` copies components and optional `@diffgazer/keys` hooks into your app and tracks them for later updates.

> **Installing via dgadd:** Components are namespaced (`ui/button`, `ui/dialog`). For the full command and flag reference — `init`, `add`, `list`, `diff`, `remove` — see the [dgadd CLI reference](/app/cli).

> **Warning:** Temporary: packages publish-gated. `@diffgazer/add`, `@diffgazer/ui`, and `@diffgazer/keys` are external publish-gated as of June 6, 2026. The package-manager commands below work only after `npm view @diffgazer/add version`, `npm view @diffgazer/ui version`, and `npm view @diffgazer/keys version` succeed. Until then, use the [pre-publication local validation](#pre-publication-local-validation) flow.

## Install (after publication)

Once `@diffgazer/add` is published, run the installer with your package manager and add a component:

```bash
pnpm dlx @diffgazer/add init
pnpm dlx @diffgazer/add add ui/button
```

```bash
npx @diffgazer/add init
npx @diffgazer/add add ui/button
```

Verify availability first with `npm view @diffgazer/add version`. Status as of June 6, 2026: not yet published — use the pre-publication flow below.

## Pre-Publication Local Validation

Until the packages are published, validate `dgadd` from a locally packed tarball.

<Steps>

<Step title="Validate the packaged flow">

Run the package smoke check from this repository:

```bash
pnpm run smoke:packages
```

</Step>

<Step title="Pack the local CLI">

To test `dgadd` in a separate fixture app, build and pack `@diffgazer/add` from this repository:

```bash
pnpm --filter @diffgazer/add build
pnpm --filter @diffgazer/add pack --pack-destination /tmp/diffgazer-packs
```

</Step>

<Step title="Install and run in the fixture app">

From the target fixture app, install the tarball and run `dgadd`:

```bash
pnpm add -D /tmp/diffgazer-packs/diffgazer-add-*.tgz
pnpm exec dgadd init
pnpm exec dgadd add ui/button
```

</Step>

</Steps>

## Setup

<Steps>

<Step title="Configure a source alias">

`dgadd init` requires a TypeScript or Vite source alias such as `@/*` or `~/*`. It rejects projects without one unless `--allow-missing-alias` is passed for an app whose tooling already resolves source aliases.

Add the alias to TypeScript:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

For Vite, mirror the same alias in `vite.config.ts`:

```ts
import { resolve } from "node:path"
import { defineConfig } from "vite"

export default defineConfig({
  resolve: {
    alias: {
      "@": resolve(__dirname, "./src"),
    },
  },
})
```

See [TypeScript](/ui/getting-started/typescript) and [Tailwind Setup](/ui/getting-started/tailwind-setup) for framework-specific details.

</Step>

<Step title="Initialize">

With the local tarball installed and the source alias in place, run `init` once and add a component:

```bash
pnpm exec dgadd init
pnpm exec dgadd add ui/button
```

This creates `diffgazer.json`, records the aliases `dgadd` uses, creates install directories, writes `src/lib/utils.ts`, and copies `src/styles/theme.css` plus `src/styles/styles.css`. It does not edit your TypeScript config, bundler config, or app CSS entrypoint.

</Step>

<Step title="Import the component CSS">

Import the copied Tailwind/theme entry from your app CSS:

```css
@import "tailwindcss";
@import "./styles/styles.css";
```

</Step>

<Step title="Add components">

Add components by namespaced item name. Bare names are rejected:

```bash
pnpm exec dgadd add ui/dialog ui/tabs ui/menu
```

</Step>

</Steps>

## Keyboard Hooks

Standalone hooks can be copied directly:

```bash
pnpm exec dgadd add keys/navigation keys/focus-trap keys/focus-restore keys/scroll-lock
```

Components with keyboard behavior choose copy mode or package mode:

```bash
pnpm exec dgadd add ui/menu --integration copy
pnpm exec dgadd add ui/menu --integration keys
```

`copy` installs local hook source. `keys` imports hooks from `@diffgazer/keys`.

## Package Managers

Use these one-shot commands only after `@diffgazer/add` is published:

```bash
pnpm dlx @diffgazer/add init
npx @diffgazer/add init
yarn dlx @diffgazer/add init
bunx @diffgazer/add init
```

## Result

After adding `ui/menu --integration copy`, a typical project looks like this:

```text
src/
  components/
    ui/
      menu/
        menu.tsx
        menu-item.tsx
        index.ts
  hooks/
    use-navigation.ts
    internal/
      navigation-dispatch.ts
  lib/
    utils.ts
  styles/
    styles.css
    theme.css
diffgazer.json
```

See [Tailwind Setup](/ui/getting-started/tailwind-setup) for theme configuration.
