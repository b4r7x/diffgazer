`@diffgazer/ui` components are installed as source files. Use `dgadd` to copy components and optional `@diffgazer/keys` hooks into your app.

> **Warning:** Temporary: packages publish-gated. `@diffgazer/add`, `@diffgazer/ui`, and `@diffgazer/keys` are external publish-gated as of May 6, 2026. The package-manager commands below work only after `npm view @diffgazer/add version`, `npm view @diffgazer/ui version`, and `npm view @diffgazer/keys version` succeed. Until then, use the [pre-publication local validation](#pre-publication-local-validation) flow.

## Install (after publication)

Once `@diffgazer/add` is published, run the installer with your package manager and add a component:

```bash
pnpm dlx @diffgazer/add init
pnpm dlx @diffgazer/add add ui/button
```

```bash
npx @diffgazer/add init
npx @diffgazer/add add ui/button
```

Verify availability first with `npm view @diffgazer/add version`. Status as of May 6, 2026: not yet published — use the pre-publication flow below.

## Pre-Publication Local Validation

Until the packages are published, validate from this repository:

```bash
pnpm run smoke:packages
```

To test `dgadd` in a separate fixture app before publication, pack and install the local CLI package:

From this repository:

```bash
pnpm --filter @diffgazer/add build
pnpm --filter @diffgazer/add pack --pack-destination /tmp/diffgazer-packs
```

From the target fixture app:

```bash
pnpm add -D /tmp/diffgazer-packs/diffgazer-add-*.tgz
pnpm exec dgadd init
pnpm exec dgadd add ui/button
```

## Required App Wiring

Configure a TypeScript or Vite source alias before running `dgadd init`, for example `@/*` or `~/*`. The initializer rejects projects without a source alias unless `--allow-missing-alias` is passed for an app whose tooling already resolves source aliases.

Add the source alias to TypeScript and to your bundler or framework:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["./src/*"]
    }
  }
}
```

For Vite, mirror the same source alias in `vite.config.ts`:

```ts
import { resolve } from "node:path"
import { defineConfig } from "vite"

export default defineConfig({
  resolve: {
    alias: {
      "@": resolve(__dirname, "./src"),
    },
  },
})
```

Then import the copied Tailwind/theme entry from your app CSS:

```css
@import "tailwindcss";
@import "./styles/styles.css";
```

See [TypeScript](/ui/getting-started/typescript) and [Tailwind Setup](/ui/getting-started/tailwind-setup) for framework-specific details.

## Initialize

After the local `@diffgazer/add` tarball is installed and the required app wiring is in place, use one command style consistently:

```bash
pnpm exec dgadd init
pnpm exec dgadd add ui/button
```

This creates `diffgazer.json`, records the aliases used by `dgadd`, creates install directories, writes `src/lib/utils.ts`, and copies `src/styles/theme.css` plus `src/styles/styles.css`. It does not edit your TypeScript config, bundler config, or app CSS entrypoint.

## Add Components

```bash
pnpm exec dgadd add ui/dialog ui/tabs ui/menu
```

Bare names are rejected. Use the namespaced item name:

```bash
pnpm exec dgadd add ui/button
```

## Add Keyboard Hooks

Standalone hooks can be copied directly:

```bash
pnpm exec dgadd add keys/navigation keys/focus-trap keys/focus-restore keys/scroll-lock
```

Components with keyboard behavior can choose copy mode or package mode:

```bash
pnpm exec dgadd add ui/menu --integration copy
pnpm exec dgadd add ui/menu --integration keys
```

`copy` mode installs local hook source. `keys` mode imports hooks from `@diffgazer/keys`.

## Package Managers

Use these package-manager one-shot commands only after `@diffgazer/add` is published:

```bash
pnpm dlx @diffgazer/add init
npx @diffgazer/add init
yarn dlx @diffgazer/add init
bunx @diffgazer/add init
```

## Result

After adding `ui/menu --integration copy`, a typical project looks like this:

```text
src/
  components/
    ui/
      menu/
        menu.tsx
        menu-item.tsx
        index.ts
  hooks/
    use-navigation.ts
    internal/
      navigation-dispatch.ts
  lib/
    utils.ts
  styles/
    styles.css
    theme.css
diffgazer.json
```

See [Tailwind Setup](/ui/getting-started/tailwind-setup) for theme configuration.
