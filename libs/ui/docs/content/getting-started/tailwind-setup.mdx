@diffgazer/ui requires Tailwind CSS 4 with the @diffgazer/ui theme variables. CSS setup depends on the consumption path:

| Path | CSS entrypoint |
|------|---------------|
| Manual copy / `dgadd` | `@import "./styles/styles.css"` (copied by `dgadd init`) |
| npm package | `@import "@diffgazer/ui/sources.css"` + `@import "@diffgazer/ui/styles.css"` |

`@diffgazer/keys` requires no CSS or Tailwind setup.

## 1. Install Tailwind 4

```bash
pnpm add -D tailwindcss @tailwindcss/vite
```

## 2. Configure Vite

```ts
// vite.config.ts
import tailwindcss from "@tailwindcss/vite"
import { defineConfig } from "vite"

export default defineConfig({
  plugins: [tailwindcss()],
})
```

## 3. Import the component CSS

If you initialized with `dgadd`, import the copied CSS entry from your app CSS:

```css
/* src/index.css */
@import "tailwindcss";
@import "./styles/styles.css";
```

`src/styles/styles.css` is the canonical component CSS entrypoint. It contains the copied theme import plus component-level CSS that copied components expect.

In the workspace source, `libs/ui/styles/styles.css` is the seed entry that imports `theme.css`. `dgadd init` and the package build materialize the canonical component entry by appending registry component CSS, such as dialog backdrop and body-lock rules, to that seed.

Use `theme.css` only when you intentionally want token variables without component CSS, for example in an advanced integration that provides its own component styles:

```css
@import "tailwindcss";
@import "./styles/theme.css";
```

`dgadd init` copies `src/styles/theme.css` and `src/styles/styles.css`, but it does not add either import to your app CSS entrypoint.

In Next App Router, import your app CSS entrypoint once from `app/layout.tsx`. Keep the Tailwind import, Diffgazer style imports, and any app-local `@source` directives in that CSS file.

## Next.js / PostCSS Setup

For Next.js projects that use PostCSS instead of Vite, install the PostCSS plugin:

```bash
pnpm add -D tailwindcss @tailwindcss/postcss
```

Create or update `postcss.config.mjs`:

```js
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
}
```

Import your app CSS entrypoint once from `app/layout.tsx`:

```tsx
import "./globals.css"
```

In `globals.css`, use the same Tailwind and Diffgazer imports as the Vite setup.

## @diffgazer/keys CSS

`@diffgazer/keys` requires no CSS or Tailwind setup. Keyboard hooks are pure React logic with no style dependencies.

## Package Mode Source Scanning

If you install `@diffgazer/ui` as a runtime npm package instead of copying source files, import the package source hints before the package styles:

> Runtime npm package mode is publish-gated until `@diffgazer/ui` and `@diffgazer/keys` are available on npm. Before that, use locally packed tarballs for validation.

```css
@import "tailwindcss";
@import "@diffgazer/ui/sources.css";
@import "@diffgazer/ui/styles.css";
```

## Theme Variables

@diffgazer/ui uses a two-layer CSS variable system:

**Layer 1 -- Primitives** (`--tui-*`): Raw color values from the Diffgazer monochrome palette.

**Layer 2 -- Semantics**: Functional tokens that reference primitives.

| Primitive | Value (dark) | Semantic tokens |
|-----------|-------------|-----------------|
| `--tui-bg` | `#0a0a0a` | `--background` |
| `--tui-fg` | `#e5e5e5` | `--foreground`, `--primary`, `--ring` |
| `--tui-dim` | `#9c9c9c` | `--muted-foreground` |
| `--tui-blue` | `#ccccff` | `--info` |
| `--tui-green` | `#e5e5e5` | `--success` |
| `--tui-red` | `#ff7b72` | `--destructive`, `--error` |
| `--tui-yellow` | `#d29922` | `--warning` |
| `--tui-violet` | `#787878` | (used in syntax highlighting) |
| `--tui-border` | `#606060` | `--border` |
| `--tui-highlight` | `#ffffff` | `--accent`, `--action` |
| `--tui-highlight-fg` | `#000000` | `--accent-foreground`, `--action-foreground` |
| `--tui-selection` | `#333333` | `--secondary`, `--card`, `--popover`, `--input` |
| `--tui-muted` | `#787878` | `--muted` |
| `--tui-input-bg` | `#0a0a0a` | Primitive available for custom inputs |

See [Theme Overview](/ui/theme/overview) for the full variable diagram.

## Yarn PnP

Package mode no longer requires consumers to hard-code a `node_modules` `@source` path. Use `@import "@diffgazer/ui/sources.css";` so package source hints travel with the published package, then validate your app's CSS tooling in a PnP workspace before relying on package mode.

## Dark Mode

Dark mode is the default. Light mode activates via `[data-theme="light"]`:

```html
<!-- Dark (default) -->
<html>

<!-- Light -->
<html data-theme="light">
```

See [Dark Mode](/ui/theme/dark-mode) for theme switching implementation.

## Font

@diffgazer/ui uses JetBrains Mono for the terminal aesthetic. Add this once in your app `<head>`:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500;700&display=swap"
/>
```

Apply the base styles to your root element:

```html
<body class="tui-base">
```
