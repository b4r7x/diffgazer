@diffgazer/ui requires Tailwind CSS 4 with the @diffgazer/ui theme variables.

## 1. Install Tailwind 4

```bash
pnpm add -D tailwindcss @tailwindcss/vite
```

## 2. Configure Vite

```ts
// vite.config.ts
import tailwindcss from "@tailwindcss/vite"

export default defineConfig({
  plugins: [tailwindcss()],
})
```

## 3. Import the theme

If you initialized with `dgadd`, import the copied CSS entry from your app CSS:

```css
/* src/index.css */
@import "./styles/styles.css";
```

`src/styles/styles.css` contains the Tailwind import and the copied theme import. If you are wiring files manually instead, use:

```css
@import "tailwindcss";
@import "./styles/theme.css";
```

`dgadd init` copies `src/styles/theme.css` and `src/styles/styles.css`, but it does not add either import to your app CSS entrypoint.

## Package Mode Source Scanning

If you install `@diffgazer/ui` as a runtime npm package instead of copying source files, add a Tailwind CSS v4 `@source` directive for the package output:

> Runtime npm package mode is publish-gated until `@diffgazer/ui` and `@diffgazer/keys` are available on npm. Before that, use locally packed tarballs for validation.

```css
@import "tailwindcss";
@source "../node_modules/@diffgazer/ui/dist";
@import "@diffgazer/ui/styles.css";
```

Adjust the `@source` path so it is relative to the CSS file that contains it.

## Theme Variables

@diffgazer/ui uses a two-layer CSS variable system:

**Layer 1 -- Primitives** (`--tui-*`): Raw color values from the GitHub Dark palette.

**Layer 2 -- Semantics**: Functional tokens that reference primitives.

| Primitive | Value (dark) | Semantic tokens |
|-----------|-------------|-----------------|
| `--tui-bg` | `#0d1117` | `--background` |
| `--tui-fg` | `#c9d1d9` | `--foreground` |
| `--tui-blue` | `#58a6ff` | `--primary`, `--accent`, `--ring`, `--info` |
| `--tui-green` | `#3fb950` | `--success` |
| `--tui-red` | `#ff7b72` | `--destructive`, `--error` |
| `--tui-yellow` | `#d29922` | `--warning` |
| `--tui-violet` | `#bc8cff` | (used in syntax highlighting) |
| `--tui-border` | `#30363d` | `--border` |
| `--tui-muted` | `#6e7681` | `--muted` |

See [Theme Overview](/ui/docs/theme/overview) for the full variable diagram.

## Dark Mode

Dark mode is the default. Light mode activates via `[data-theme="light"]`:

```html
<!-- Dark (default) -->
<html>

<!-- Light -->
<html data-theme="light">
```

See [Dark Mode](/ui/docs/theme/dark-mode) for theme switching implementation.

## Font

@diffgazer/ui uses JetBrains Mono for the terminal aesthetic. Add this once in your app `<head>`:

```html
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin />
<link
  rel="stylesheet"
  href="https://fonts.googleapis.com/css2?family=JetBrains+Mono:wght@400;500;700&display=swap"
/>
```

Apply the base styles to your root element:

```html
<body class="tui-base">
```
