# RDY-002: shadcn Registry Schema And Keys Namespace Are Not Install-Safe

Area: Registry and shadcn installability

Severity: P0

Effort: M

## Problem

The keys registry output does not cleanly match the shadcn registry schema or namespace-resolution expectations. This can break external registry consumers even if the hosted endpoint becomes reachable.

## Evidence

- `libs/keys/public/r/navigation.json:16` and `libs/keys/public/r/navigation.json:18` emit `registry:internal`, which is not a supported public shadcn registry item type.
- `libs/keys/public/r/navigation.json:12` imports `../internal/navigation-dispatch.js`, while the local copy-bundle path rewrite expects a different layout.
- `cli/add/scripts/generate-keys-copy-bundle.ts:9-11` rewrites keys internals for copy mode, which means hosted-registry JSON and CLI-copy JSON have different path assumptions.
- `libs/ui/docs/content/utils/shadcn-namespace.mdx:39` warns about namespace validation, but installability is not proven against the public schema and hosted source.

## User Impact

Users can get registry JSON that validates locally but fails in shadcn tooling or resolves internal keys paths differently depending on whether they install from the hosted registry or the custom CLI.

## Fix

Make keys registry items schema-valid and path-stable across hosted and CLI-copy modes.

Concrete fix:

- Stop emitting unsupported `registry:internal` in public registry JSON.
- Represent internal files using supported file metadata, or keep them as private implementation files inside the installed item.
- Ensure hosted registry paths and CLI copy-bundle paths are generated by the same resolver.
- Validate public JSON against the official shadcn schema in CI.

## Acceptance Criteria

- Every public `libs/keys/public/r/*.json` item validates against the shadcn registry item schema.
- `keys:navigation` installs from hosted registry and custom CLI without path rewrites diverging.
- Installed keys files import their internal helpers through paths that exist in the clean consumer tree.
- Namespace docs reflect the exact supported install syntax.

## Verification

In a clean project:

- Fetch `https://keys.diffgazer.com/r/navigation.json`.
- Validate it against `https://ui.shadcn.com/schema/registry-item.json`.
- Install it through the intended shadcn flow.
- Install it through `dgadd`.
- Run `tsc --noEmit` and verify the installed keys hook imports resolve.
