# Issues

Each issue captures one implementation problem area from the 20-agent audit.

Priority meanings:

- P0: broken runtime behavior, wrong public API, or handoff blocker.
- P1: important correctness, accessibility, docs, or maintenance blocker for SOTA 5/5.
- P2: polish needed before claiming 5/5, but not usually a standalone runtime blocker.

Treat issue details as audit findings to verify against the current tree. All remaining P0/P1/P2 items must be resolved or explicitly documented as deferred with a defensible reason before final `$sota-verify`.
