# Issues

Priority definitions:

- `P0`: public install, local tarball, package import, copy-first, or shadcn-style install breaks.
- `P1`: user-facing API, docs, accessibility, behavior, or release contract is inconsistent.
- `P2`: quality hardening, confidence gap, maintainability, or polish.
