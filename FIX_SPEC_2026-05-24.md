#### [TASK-005] Timing-safe token comparison
**Findings**: SEC-005 (Audit), R2-SEC-001 (Opus)
**Files**:
- `cli/server/src/app.ts` (~line 85)
- `cli/server/src/features/shutdown/router.ts` (~line 9)
**Fix**: Replace string `!==` comparisons with `crypto.timingSafeEqual`:
```ts
import { timingSafeEqual } from "node:crypto";
function safeTokenMatch(header: string | undefined, expected: string): boolean {
  if (!header || header.length !== expected.length) return false;
  return timingSafeEqual(Buffer.from(header), Buffer.from(expected));
}
```
Apply this function in both `app.ts` auth middleware and `shutdown/router.ts` token check.
**Effort**: Low
**Parallel**: yes
**Verification**: Existing tests pass; add a test asserting equal-length wrong token is rejected
