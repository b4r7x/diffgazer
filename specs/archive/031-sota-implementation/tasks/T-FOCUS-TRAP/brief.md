# Task T-FOCUS-TRAP — Fix useFocusTrap to document-level capture + focusin recapture

**Phase:** 2
**Blocks:** none
**Blocked by:** none

## Goal
`libs/keys/src/hooks/use-focus-trap.ts:102` attaches `keydown` to the CONTAINER in BUBBLE phase. Consequences:
- Focus escaped outside container → next Tab is not intercepted (listener never fires).
- Descendant calls `event.stopPropagation()` on keydown → trap bypassed silently.
- Cross-iframe / shadow DOM → propagation retargeting issues.
- No-focusable-children + container without `tabIndex` → `pickInitialTarget()` falls back to container, `.focus()` is no-op, listener never receives Tab.

Pick one option:

**Option A (preferred):** Implement full document-level capture trap matching Radix/react-aria/focus-trap npm:
- `container.ownerDocument.addEventListener("keydown", handleKeyDown, true)` (capture phase)
- `container.ownerDocument.addEventListener("focusin", handleFocusIn, true)` (recapture)
- `MutationObserver` for removed-focused-element handling
- Track `lastFocusedRef` so recapture lands on last in-trap element
- Tests for: focus-escaped→Tab→recapture; stopPropagation→Tab still wraps; cross-iframe; no-tabbable+escaped

**Option B:** Rename to `useFocusBoundary` and document that it only wraps Tab when focus is already inside. Update README, MDX, registry. No code change to behavior. This is honest but ships a weaker primitive than the name implies.

Recommend Option A. If too risky, do Option B and file Option A as follow-up.
