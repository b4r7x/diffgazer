Composable, scoped keyboard navigation hooks for React 19.

- **Zero dependencies**, ~1.5 KB gzipped
- **Scope stack** -- only the topmost scope receives events, no conflicts between layers
- **React 19 only** -- uses `useEffectEvent` for stable handlers without stale closures
- **ESM only**

```bash
npm install keyscope
```

## Installation Options

diffui components embed keyboard hooks internally â€” like Radix UI embeds roving-focus, focus traps, and scroll locks. You have four ways to source these hooks:

### Option 1: shadcn registry (automatic)

When you install a diffui component that requires keyboard hooks via the shadcn CLI, the hooks are installed automatically:

```bash
npx shadcn@latest add @diffui/dialog   # installs use-focus-trap + use-scroll-lock
npx shadcn@latest add @diffui/menu     # installs use-navigation + use-focus-trap + use-scroll-lock
```

This works via `registryDependencies` â€” diffui component entries declare which keyscope hooks they depend on, and the shadcn CLI resolves and installs them from your configured `@keyscope` registry namespace (for example `https://<your-diffgazer-domain>/r/keyscope/{name}.json`). No separate step needed.

You can also install hooks standalone:

```bash
npx shadcn@latest add @keyscope/navigation
npx shadcn@latest add @keyscope/focus-trap
npx shadcn@latest add @keyscope/scroll-lock
```

### Option 2: Install keyscope package

```bash
npm install keyscope
# or via diffui CLI:
npx diffui add menu --integration keyscope
```

Components import hooks directly from the keyscope package. Gives you the full API including `KeyboardProvider`, `useKey`, `useScope`, and `useFocusZone`.

### Option 3: Copy standalone hooks

```bash
# Via keyscope CLI (recommended):
npx keyscope add navigation focus-trap scroll-lock

# Or via diffui CLI:
npx diffui add menu --integration copy
```

Copies three standalone hooks into your project at `@/hooks/`. No keyscope dependency required. Components import from local paths:

```tsx
import { useNavigation } from "@/hooks/use-navigation"
import { useFocusTrap } from "@/hooks/use-focus-trap"
import { useScrollLock } from "@/hooks/use-scroll-lock"
```

### Option 4: Manual copy-paste

Copy the hook source code from the [Hook Source Code](#hook-source-code) section below into your project.

## Quick Start

Wrap your app with `KeyboardProvider`, then use hooks anywhere inside it:

```tsx
import { KeyboardProvider, useKey } from "keyscope"

function App() {
  return (
    <KeyboardProvider>
      <Editor />
    </KeyboardProvider>
  )
}

function Editor() {
  useKey("mod+s", (e) => {
    e.preventDefault()
    save()
  })

  useKey("Escape", () => close())

  return <div>...</div>
}
```

`mod` maps to `Meta` on Mac and `Ctrl` on Windows/Linux.

## Hooks Reference

| Hook | Purpose | Provider Required | Docs |
|------|---------|:-:|------|
| `useNavigation` | DOM-driven list/tab navigation (arrow keys, Home/End, Enter) | No | [Navigation](/docs/integrations/keyscope-navigation) |
| `useFocusTrap` | Trap Tab/Shift+Tab within a container | No | [Utilities](/docs/integrations/keyscope-utilities) |
| `useScrollLock` | Prevent scrolling (ref-counted) | No | [Utilities](/docs/integrations/keyscope-utilities) |
| `useKey` | Bind keyboard shortcuts (single, array, or key map) | Yes* | [useKey](/docs/integrations/keyscope-usekey) |
| `useScope` | Push a named scope onto the stack | Yes | [useKey](/docs/integrations/keyscope-usekey) |
| `useFocusZone` | Multi-zone keyboard navigation with transitions | Yes | [Focus Zones](/docs/integrations/keyscope-focus) |
| `keys()` | Map multiple hotkeys to the same handler | No | [Utilities](/docs/integrations/keyscope-utilities) |

\* `useKey` silently does nothing without a provider â€” safe to use in portable components.

**Standalone hooks** (`useNavigation`, `useFocusTrap`, `useScrollLock`) work without `KeyboardProvider` and are available in copy mode. The rest require the full keyscope package.

## Patterns with diffui

diffui components embed keyscope hooks (`useNavigation`, `useFocusTrap`, `useScrollLock`) internally for WCAG-compliant keyboard navigation. Components also expose `highlighted`, `onHighlightChange`, and `onKeyDown` props for external control. See the [Patterns](/docs/integrations/keyscope-patterns) page for complete integration examples with Menu, Dialog, CommandPalette, Tabs, and multi-zone layouts.
