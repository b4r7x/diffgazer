---
title: diffui add
description: Add components to your project. Copies source files directly — no hidden dependencies, no runtime package.
---

> **shadcn users**: If you already use shadcn/ui, you can install diffui components via `npx shadcn@latest add @diffui/<component>` instead. See [Installation](/docs/getting-started/installation#option-b-shadcn-registry) for setup.

## Usage

```bash
npx diffui add <component> [component...]
```

## Examples

```bash
# Add a single component
npx diffui add button

# Add multiple components
npx diffui add dialog tabs menu

# Add all components
npx diffui add --all

# Add keyboard integrations as local copied hooks (no keyscope dependency)
npx diffui add menu --integration copy

# Add keyboard integrations backed by keyscope
npx diffui add menu --integration keyscope

# Pin keyscope version used for copy/keyscope modes
npx diffui add menu --integration copy --keyscope-version 0.1.1
```

## What It Does

1. Reads the component entry from `registry.json`
2. Copies all component files to your `components/ui/` directory
3. Installs any missing npm dependencies
4. Resolves registry dependencies (e.g., `stepper` depends on `badge`)
5. Optionally installs keyboard integration hooks (`copy` or `keyscope` mode)

## Options

| Flag | Description | Default |
|------|-------------|---------|
| `--cwd <path>` | Working directory | `.` |
| `--all` | Add all components | `false` |
| `--overwrite` | Overwrite existing files | `false` |
| `--dry-run` | Preview changes without writing files | `false` |
| `--integration <mode>` | Keyboard integration mode: `ask \| none \| copy \| keyscope` | `ask` |
| `--keyscope-version <version>` | Version/tag used by `copy` and `keyscope` modes | `latest` |
| `--yes` | Skip confirmation prompts | `false` |

## Integration Modes

- `none`: no keyboard integration files are added
- `copy`: CLI installs `keyscope@<version>` in a temporary directory and copies hooks from `keyscope`'s published registry (`registry/registry.json`) into your local hooks folder
- `keyscope`: same copied local hooks as `copy` mode, plus installs `keyscope` in your project dependencies

## Copied Keyboard Hooks

When integration mode is `copy` or `keyscope`, the CLI writes these files into your hooks folder:

- `use-navigation.ts`
- `use-focus-trap.ts`
- `use-scroll-lock.ts`

Use them from your app code:

```tsx
import { useNavigation } from "@/hooks/use-navigation"
import { useFocusTrap } from "@/hooks/use-focus-trap"
import { useScrollLock } from "@/hooks/use-scroll-lock"
```

## Registry Dependencies

Some components depend on other diffui components. These are installed automatically:

| Component | Depends On |
|-----------|-----------|
| `stepper` | `badge` |
| `code-block` | `scroll-area` |
| `toggle-group` | `button` |

## File Structure

Each component is copied as its full source directory:

```bash
$ npx diffui add dialog

  Adding dialog...
    src/components/ui/dialog/dialog.tsx
    src/components/ui/dialog/dialog-trigger.tsx
    src/components/ui/dialog/dialog-content.tsx
    src/components/ui/dialog/dialog-context.tsx
    src/components/ui/dialog/dialog-header.tsx
    src/components/ui/dialog/dialog-title.tsx
    src/components/ui/dialog/dialog-description.tsx
    src/components/ui/dialog/dialog-body.tsx
    src/components/ui/dialog/dialog-footer.tsx
    src/components/ui/dialog/dialog-close.tsx
    src/components/ui/dialog/index.ts

  Done.
```

> **Tip**: Components are plain React source files. You own them — edit, extend, or delete as needed.

## Shadcn Alternative

If you prefer to use shadcn/ui tooling, diffui components are available via the shadcn CLI. Keyboard hooks are resolved and installed automatically — no `--integration` flag needed. See [Installation](/docs/getting-started/installation#option-b-shadcn-registry) for the namespace registry setup.
