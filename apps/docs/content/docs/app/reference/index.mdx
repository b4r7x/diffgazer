This is the look-it-up section: the flags, file fields, provider names, output shapes, and server behavior Diffgazer commits to, written to be checked rather than read end to end.

## In this section

- [CLI](/app/reference/cli) — every command and flag for the `diffgazer` binary, including the `--tui` mode switch.
- [Configuration](/app/reference/configuration) — the config file Diffgazer reads, field by field, with defaults and accepted values.
- [Providers](/app/reference/providers) — the model providers Diffgazer can talk to and how bring-your-own-key fits each one.
- [Review output](/app/reference/review-output) — the shape of a finished review: how issues are grouped by severity and category.
- [Environment variables](/app/reference/environment-variables) — the variables Diffgazer reads, what each controls, and how they override config.
- [Server](/app/reference/server) — the local server web mode runs against: its lifecycle, the shutdown token, and how a session ends.
