The fastest path to a review is three commands, run from inside a git repository that has uncommitted changes. You also need a key for one supported provider before diffgazer will let a review start.

## Prerequisites

- A git repository with uncommitted changes to review.
- Node.js 20 or newer.
- An API key for one provider: Gemini, Z.AI, Z.AI Coding Plan, OpenRouter, Groq, or Cerebras.

## Steps

<Steps>

<Step title="Install the CLI">

```bash
npm install -g diffgazer
```

</Step>

<Step title="Change into your repo">

```bash
cd your-project
```

</Step>

<Step title="Start diffgazer">

```bash
diffgazer
```

It launches a local server and opens the web environment in your browser.

</Step>

<Step title="Complete onboarding">

Pick a provider, paste your key, choose a model, and trust the repo.

</Step>

<Step title="Start a review">

Start a review from the web environment.

</Step>

<Step title="Read the findings">

Read the findings inline against your diff.

</Step>

</Steps>

> Tip: When you are done, stop the server with `Ctrl+C` in the terminal where diffgazer is running.

## Readiness checklist

diffgazer blocks a review until setup is complete and returns a clear list of what is still missing. Onboarding walks you through each item, so run through this list before you start your first review:

- Node.js 20 or newer is installed.
- `git` is installed and you are inside a git repository.
- The repository has changes to review.
- A secrets storage backend is chosen during onboarding (your OS keyring or an encrypted file).
- A provider is active, its API key is saved and readable, and a model is selected.
- The current directory is trusted so diffgazer can read its files.

If any item is missing, finish it in onboarding and run the review again.
