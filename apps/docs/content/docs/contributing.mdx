## Development Setup

```bash
git clone https://github.com/diffgazer/diffui.git
cd diffui
pnpm install
```

## Adding a Component

1. Create component files in `registry/ui/<name>/`
2. Add entry to `registry/registry.json`
3. Create 2-3 demo files in `registry/examples/<name>/`
4. Add `registry/component-docs/<name>.ts` (required for component docs rendering)

The docs pipeline generates component data (props/source/examples) and component MDX pages with TOC anchors from the registry entry.

Generated artifacts are produced from the repo sources:

- `public/r/*`
- `docs/generated/*`
- `docs/content/components/*`

## Building Docs Artifacts

```bash
# Optional: override registry host used inside generated JSON files
# export REGISTRY_ORIGIN="https://<your-diffgazer-domain>"

pnpm build:docs
```

## Verification Before PR

```bash
pnpm verify:components
pnpm --filter @diffui/scripts build-docs-data
```

## Code Style

- ESM imports only (no CommonJS)
- Tailwind 4 + CVA for component variants
- JetBrains Mono font, terminal aesthetic
- No manual useCallback/useMemo (React 19 Compiler handles this)
- Use `cn()` from `@/lib/utils` for class merging

## Pull Requests

- One component per PR (unless they're tightly coupled)
- Include demo files with the component
- Ensure TypeScript compiles cleanly
- Test at 320px and 1440px widths
