## Overview

The diffgazer CLI ships with a purpose-built terminal component library. Every component is a React component rendered via [Ink 6](https://github.com/vadimdemedes/ink), targeting Node.js terminal emulators rather than browsers. The library mirrors the structure and naming of the web UI's diff-ui components so that both surfaces stay conceptually aligned.

The library provides 18 UI components, 3 layout components, a theme system with 3 palettes, a scope-based keyboard manager, and a stack-based screen router.

## Architecture

- **Ink 6 + React 19** -- components use JSX and hooks, rendered to the terminal via Ink's reconciler.
- **Theme system** -- all color values flow through `useTheme()`, which reads from `CliThemeProvider`. Three palettes ship by default (dark, light, high-contrast). Terminal background is auto-detected via `COLORFGBG`.
- **Keyboard management** -- `TerminalKeyboardProvider` maintains a scope stack. Screens push/pop scopes; hotkeys are dispatched only to the active scope. Global shortcuts (q, s, ?) bypass the scope stack.
- **Screen router** -- a simple `NavigationProvider` with `navigate(route)` / `goBack()`. Routes are a discriminated union; the router maps each `ScreenName` to a component.

## Web-to-Terminal Mapping

The terminal components are not 1:1 ports of their web counterparts. Terminal constraints (no mouse, monospace grid, 256-color palette) require different trade-offs:

| Web (diff-ui) | Terminal (CLI) | Key Difference |
|---------------|---------------|----------------|
| CSS colors + Tailwind tokens | `CliColorTokens` via `useTheme()` | All colors are terminal-safe hex strings |
| Click handlers | `useInput` / Enter key | No mouse; keyboard-only interaction |
| CSS layout (flex/grid) | Ink `<Box>` flex model | Subset of flexbox; no grid, no absolute positioning |
| Hover states | Highlight index | Arrow keys move highlight; no hover concept |
| Portals / z-index | Full-screen `<Box>` overlay | Dialog fills the entire terminal viewport |
| Toast via DOM portal | Toast via `useSyncExternalStore` | Module-level store; `toast()` callable from anywhere |

| Layer | Technology |
|-------|-----------|
| Runtime | Node.js |
| Rendering | Ink 6.8.0 |
| UI framework | React 19.2.4 |
| Spinner | ink-spinner 5.0.0 |

| Module format | ESM only (.js extensions) |
