## Overview

Footer renders a horizontal bar of keyboard shortcut hints at the bottom of the terminal. Shortcuts are displayed as `[key] label` pairs. An optional right-aligned group can show secondary shortcuts separated by a vertical bar.

## Import

```tsx
import { Footer } from "../../components/layout/footer.js";
import type { Shortcut } from "../../components/layout/footer.js";
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `shortcuts` | `Shortcut[]` | -- | Left-aligned shortcut hints |
| `rightShortcuts` | `Shortcut[]` | -- | Right-aligned shortcut hints (optional) |

### Shortcut Type

```ts
interface Shortcut {
  key: string;     // Key label (e.g., "Enter", "q", "Esc")
  label: string;   // Description (e.g., "Select", "Quit", "Back")
  disabled?: boolean;
}
```

The `Shortcut` type is defined in `@diffgazer/schemas/ui`.

## Usage

```tsx
const shortcuts: Shortcut[] = [
  { key: "Enter", label: "Select" },
  { key: "Esc", label: "Back" },
];

const rightShortcuts: Shortcut[] = [
  { key: "q", label: "Quit" },
];

<Footer shortcuts={shortcuts} rightShortcuts={rightShortcuts} />
```

Renders:

```
────────────────────────────────────────────────────────────────────────────────
[Enter] Select  [Esc] Back                                     │ [q] Quit
```

## Layout

- An 80-character horizontal rule is drawn above the shortcut hints.
- Left shortcuts are spaced with `gap={2}`.
- Right shortcuts appear after a vertical bar separator, aligned to the right.
- Key labels use the `accent` token in bold. Descriptions use the `muted` token.

## Notes

- Each screen sets its own shortcuts via the `usePageFooter` hook, which updates the `FooterProvider` context.
- This component is typically not used directly; it is rendered by `GlobalLayout`.
- The horizontal rule and separator colors use the `border` token.
