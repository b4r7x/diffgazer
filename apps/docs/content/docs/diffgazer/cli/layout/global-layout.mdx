## Overview

GlobalLayout is the root layout component for the CLI app. It fills the entire terminal viewport and arranges three regions vertically: the Header at the top, a flex-grow content area in the middle, and the Footer at the bottom.

## Import

```tsx
import { GlobalLayout } from "../../components/layout/global-layout.js";
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Screen content rendered in the center region |

## Usage

GlobalLayout is mounted once in the `App` component and wraps the `ScreenRouter`:

```tsx
<GlobalLayout>
  <ScreenRouter />
</GlobalLayout>
```

## Internal Wiring

GlobalLayout reads from two context providers:

1. **NavigationProvider** -- provides `goBack` and `canGoBack` to control the Header's back indicator.
2. **FooterProvider** -- provides `shortcuts` and `rightShortcuts` for the Footer bar.

```tsx
export function GlobalLayout({ children }: GlobalLayoutProps) {
  const { stdout } = useStdout();
  const { goBack, canGoBack } = useNavigation();
  const { shortcuts, rightShortcuts } = useFooterContext();

  return (
    <Box flexDirection="column" width={stdout.columns} height={stdout.rows}>
      <Header
        providerName="diffgazer"
        providerStatus="idle"
        onBack={canGoBack ? goBack : undefined}
      />
      <Box flexGrow={1} flexDirection="column">
        {children}
      </Box>
      <Footer shortcuts={shortcuts} rightShortcuts={rightShortcuts} />
    </Box>
  );
}
```

## Viewport Sizing

The layout reads `stdout.columns` and `stdout.rows` from Ink's `useStdout()` hook to fill the entire terminal window. If the terminal is resized, Ink re-renders and the layout adjusts.

## Provider Nesting

For GlobalLayout to function, the component tree must include (in order from outermost to innermost):

1. `CliThemeProvider`
2. `TerminalKeyboardProvider`
3. `NavigationProvider`
4. `FooterProvider`
5. `GlobalLayout`

This nesting is handled in the `App` component at `apps/cli/src/app/index.tsx`.

## Notes

- The Header always displays "diffgazer" as the provider name with "idle" status. This is a placeholder; future versions may display the active AI provider.
- The content area has `flexGrow={1}` so it expands to fill all space between Header and Footer.
