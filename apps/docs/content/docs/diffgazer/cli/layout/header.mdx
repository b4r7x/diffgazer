## Overview

Header renders the top bar of the CLI interface. It shows the current provider name with a connection status dot on the left, and the "diffgazer" brand name on the right. When back navigation is available, a back indicator appears before the provider name.

## Import

```tsx
import { Header } from "../../components/layout/header.js";
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `providerName` | `string` | -- | Name of the active AI provider |
| `providerStatus` | `"active" \| "idle"` | -- | Connection status; `active` shows green dot, `idle` shows muted dot |
| `onBack` | `() => void` | -- | If provided, shows a `[<]` back indicator before the provider name |

## Usage

```tsx
<Header providerName="Gemini" providerStatus="active" />
```

Renders:

```
Gemini ●                                    ┃ diffgazer
```

With back navigation:

```tsx
<Header providerName="Gemini" providerStatus="active" onBack={goBack} />
```

```
[<] Gemini ●                                ┃ diffgazer
```

## Layout

The header uses `justifyContent="space-between"` to push the provider info to the left and the brand name to the right. It spans the full terminal width.

## Notes

- The status dot color uses `tokens.green` for active and `tokens.muted` for idle.
- The brand name "diffgazer" is rendered in bold with the `accent` token color.
- The vertical bar separator uses the `border` token color.
- This component is typically not used directly; it is rendered by `GlobalLayout`.
