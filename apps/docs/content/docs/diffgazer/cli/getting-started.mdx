The CLI app lives at `apps/cli/` within the diffgazer monorepo. Terminal components are organized as follows:

```

      scroll-area.tsx
      section-header.tsx
      spinner.tsx
      tabs.tsx
      toast.tsx

      status-display.tsx
    layout/           -- App shell components
      header.tsx
      footer.tsx
      global-layout.tsx
  theme/
    palettes.ts       -- CliColorTokens + 3 palette definitions
    theme-context.tsx  -- CliThemeProvider + useTheme hook
  hooks/
    use-scope.ts      -- Push/pop keyboard scope
    use-key.ts        -- Register scoped hotkey handler
    use-page-footer.ts -- Set footer shortcuts for current screen
    use-back-handler.ts -- Escape-to-go-back handler
  app/
    routes.ts         -- Route discriminated union
    navigation-context.tsx -- NavigationProvider + useNavigation
    router.tsx        -- ScreenRouter mapping
    providers/
      keyboard-provider.tsx -- TerminalKeyboardProvider
      footer-provider.tsx   -- Footer shortcut state
    screens/          -- Screen components
  features/           -- Feature modules (home, review, settings, etc.)
  types/
    components.ts     -- Shared Variant and Size types
    cli.ts            -- CliMode type
```

## Key Patterns

### Every Component Uses `useTheme()`

All color values come from the theme context. Never hardcode color strings.

```tsx
import { useTheme } from "../../theme/theme-context.js";

export function MyComponent() {
  const { tokens } = useTheme();
  return <Text color={tokens.accent}>themed text</Text>;
}
```

### Interactive Components Accept `isActive`

Components that listen for keyboard input accept an `isActive` prop that gates `useInput`. This prevents multiple components from capturing the same keystrokes. Pass `isActive={true}` only to the component that should currently own focus.

```tsx
<Menu isActive={currentPane === "menu"} onSelect={handleSelect}>
  <Menu.Item id="a">Alpha</Menu.Item>
  <Menu.Item id="b">Beta</Menu.Item>
</Menu>
```

### Compound Components

Several components use the compound component pattern with `Object.assign`:

- `Callout` -- `Callout.Title`, `Callout.Content`
- `CheckboxGroup` -- `CheckboxGroup.Item`
- `Dialog` -- `Dialog.Content`, `Dialog.Header`, `Dialog.Title`, `Dialog.Body`, `Dialog.Footer`
- `EmptyState` -- `EmptyState.Message`, `EmptyState.Description`
- `Menu` -- `Menu.Item`, `Menu.Divider`
- `NavigationList` -- `NavigationList.Item`, `NavigationList.Title`, `NavigationList.Badge`, `NavigationList.Status`, `NavigationList.Subtitle`
- `Panel` -- `Panel.Header`, `Panel.Content`, `Panel.Footer`
- `RadioGroup` -- `RadioGroup.Item`
- `Tabs` -- `Tabs.List`, `Tabs.Trigger`, `Tabs.Content`

## Adding a New Component

1. Create the file in `apps/cli/src/components/ui/your-component.tsx`.
2. Export the props interface and the component function.
3. Use `useTheme()` for all colors.
4. If the component handles keyboard input, accept an `isActive` prop and pass it to `useInput`.
5. Use ESM imports with `.js` extensions.

```tsx
import type { ReactNode } from "react";
import { Box, Text } from "ink";
import { useTheme } from "../../theme/theme-context.js";

export interface YourComponentProps {
  children: ReactNode;
}

export function YourComponent({ children }: YourComponentProps) {
  const { tokens } = useTheme();
  return (
    <Box borderStyle="round" borderColor={tokens.border}>
      <Text color={tokens.fg}>{children}</Text>
    </Box>
  );
}
```

## Adding a New Screen

1. Create the screen file in `apps/cli/src/app/screens/`.
2. Add the route variant to the `Route` union in `apps/cli/src/app/routes.ts`.
3. Register the component in `SCREEN_MAP` in `apps/cli/src/app/router.tsx`.
4. If the screen needs a deterministic back target, add an entry in `apps/cli/src/lib/back-navigation.ts`.

Every screen should call three hooks at the top:

```tsx
import { useScope } from "../../hooks/use-scope.js";
import { usePageFooter } from "../../hooks/use-page-footer.js";
import { useBackHandler } from "../../hooks/use-back-handler.js";

export function MyScreen(): ReactElement {
  useScope("my-screen");          // keyboard scope
  usePageFooter({ shortcuts: MY_SHORTCUTS }); // footer hints
  useBackHandler();               // Escape goes back

  // ...
}
```

### Screen Convention Summary

| Hook | Purpose |
|------|---------|
| `useScope(name)` | Pushes a keyboard scope on mount, pops on unmount |
| `usePageFooter(opts)` | Sets the footer shortcut hints for this screen |
| `useBackHandler()` | Binds Escape to `goBack()` from NavigationProvider |

## Running the CLI

```bash
# Development mode (hot-reload, separate servers)
cd apps/cli && pnpm dev

# Production mode (embedded server + bundled web)
pnpm build && pnpm start
```

The `--theme` flag overrides the auto-detected palette:

```bash
diffgazer --theme dark
diffgazer --theme light
diffgazer --theme high-contrast
```
