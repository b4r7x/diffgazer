## Overview

NavigationList is a compound component for building navigable lists with rich item content. Each item can contain a title, badge, status indicator, and subtitle. It supports selected and highlighted states, making it suitable for sidebar navigation and list-detail layouts.

## Web Equivalent

Maps to diff-ui's `NavigationList` component. The terminal version uses `>` and `|` prefixes instead of CSS selection indicators.

## Import

```tsx
import { NavigationList } from "../../components/ui/navigation-list.js";
```

## Props

### NavigationList

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `selectedId` | `string \| null` | `null` | Currently selected item id (shown with `\|` prefix) |
| `highlightedId` | `string \| null` | `null` | Controlled highlight; overrides internal tracking |
| `onSelect` | `(id: string) => void` | -- | Called when an item is selected (Enter key) |
| `onHighlightChange` | `(id: string) => void` | -- | Called when the highlighted item changes |
| `focused` | `boolean` | -- | Reserved for future use |
| `wrap` | `boolean` | `true` | Wrap navigation from last to first item |
| `isActive` | `boolean` | `true` | Whether the list accepts keyboard input |
| `children` | `ReactNode` | -- | Should contain `NavigationList.Item` elements |

### NavigationList.Item

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `id` | `string` | -- | Unique identifier |
| `disabled` | `boolean` | `false` | Disable this item |
| `children` | `ReactNode` | -- | Item content (can include sub-components) |

### NavigationList.Title

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Bold title text |

### NavigationList.Badge

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `string` | -- | Color variant (`"success"`, `"warning"`, `"error"`, `"info"`, or any other string for muted) |
| `children` | `ReactNode` | -- | Badge label rendered inside brackets |

### NavigationList.Status

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Status text in muted color |

### NavigationList.Subtitle

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Subtitle text in muted color |

## Usage

```tsx
<NavigationList onSelect={handleSelect} selectedId={activeId}>
  <NavigationList.Item id="issue-1">
    <NavigationList.Title>Missing null check</NavigationList.Title>
    <NavigationList.Badge variant="error">high</NavigationList.Badge>
    <NavigationList.Status>parser.ts:42</NavigationList.Status>
  </NavigationList.Item>
  <NavigationList.Item id="issue-2">
    <NavigationList.Title>Unused import</NavigationList.Title>
    <NavigationList.Badge variant="info">low</NavigationList.Badge>
    <NavigationList.Subtitle>Consider removing</NavigationList.Subtitle>
  </NavigationList.Item>
</NavigationList>
```

## Visual States

Items have three visual states indicated by their prefix:

| State | Prefix | Style |
|-------|--------|-------|
| Normal | `  ` (2 spaces) | Default color |
| Highlighted | `> ` | Bold, accent background |
| Selected | `\| ` | Bold, accent foreground |

## Keyboard Interaction

| Key | Action |
|-----|--------|
| Up Arrow | Move highlight to previous item |
| Down Arrow | Move highlight to next item |
| Enter | Select the highlighted item |

## Notes

- Disabled items are rendered dimmed and skipped during navigation.
- Sub-components (`Title`, `Badge`, `Status`, `Subtitle`) are rendered inline within the item's content area. Layout depends on how they are composed.
- The `focused` prop is accepted but currently unused (reserved for future split-pane focus management).
