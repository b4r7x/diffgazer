## Overview

Dialog is a compound component that renders a centered overlay filling the entire terminal viewport. It is conditionally rendered based on an `open` prop and dismisses on Escape. Use it for confirmations, forms, and detail views that need to overlay the current screen.

## Web Equivalent

Maps to diff-ui's `Dialog` component. The terminal version renders a full-screen `<Box>` instead of using CSS portals or z-index. There is no backdrop blur -- the dialog simply replaces the visible content.

## Import

```tsx
import { Dialog } from "../../components/ui/dialog.js";
```

## Props

### Dialog

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `open` | `boolean` | `false` | Whether the dialog is visible |
| `onOpenChange` | `(open: boolean) => void` | -- | Called when the dialog requests to close (Escape key) |
| `children` | `ReactNode` | -- | Should contain `Dialog.Content` |

### Dialog.Content

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Dialog content, typically containing Header, Body, and Footer |

### Dialog.Header

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Header content with bottom border separator |

### Dialog.Title

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Bold title text |

### Dialog.Body

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Main content area (flex-grow) |

### Dialog.Footer

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `ReactNode` | -- | Footer content with top border separator |

## Usage

```tsx
const [open, setOpen] = useState(false);

<Dialog open={open} onOpenChange={setOpen}>
  <Dialog.Content>
    <Dialog.Header>
      <Dialog.Title>Confirm Action</Dialog.Title>
    </Dialog.Header>
    <Dialog.Body>
      <Text>Are you sure you want to proceed?</Text>
    </Dialog.Body>
    <Dialog.Footer>
      <Button isActive={open} onPress={() => setOpen(false)}>Cancel</Button>
    </Dialog.Footer>
  </Dialog.Content>
</Dialog>
```

## Keyboard Interaction

| Key | Action |
|-----|--------|
| Escape | Calls `onOpenChange(false)` to request closing the dialog |

## Layout

The dialog uses the terminal's `stdout.columns` and `stdout.rows` to size itself to fill the entire viewport. Content is centered both horizontally and vertically. `Dialog.Content` adds a rounded border, horizontal padding of 2, and vertical padding of 1.

## Notes

- When `open` is `false`, the component renders `null`.
- `Dialog.Header` and `Dialog.Footer` render single-line border separators (bottom and top respectively).
- The Escape handler is only active when `open` is `true`.
