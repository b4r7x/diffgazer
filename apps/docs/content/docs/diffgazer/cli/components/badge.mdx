## Overview

Badge renders a bracketed inline label colored by semantic variant. Use it for status indicators, severity labels, and category tags. The bracket style (`[label]`) is a terminal convention for visually distinct inline markers.

## Web Equivalent

Maps to diff-ui's `Badge` component. The terminal version uses bracket delimiters instead of rounded pill shapes and derives colors from `CliColorTokens` rather than Tailwind classes.

## Import

```tsx
import { Badge } from "../../components/ui/badge.js";
```

## Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `variant` | `"success" \| "warning" \| "error" \| "info" \| "neutral"` | `"neutral"` | Semantic color variant |
| `size` | `"sm" \| "md" \| "lg"` | `"sm"` | Internal padding around the label |
| `dot` | `boolean` | `false` | Prepend a filled circle indicator before the label |
| `children` | `ReactNode` | -- | Label content |

## Usage

```tsx
<Badge variant="success">PASS</Badge>
```

Renders: `[PASS]` in the theme's success color.

## Variants

```tsx
<Badge variant="success">passed</Badge>    // green
<Badge variant="warning">pending</Badge>   // yellow
<Badge variant="error">failed</Badge>      // red
<Badge variant="info">running</Badge>      // blue
<Badge variant="neutral">unknown</Badge>   // muted
```

## Sizes

Size controls the padding inside the brackets:

```tsx
<Badge size="sm">sm</Badge>    // [sm]
<Badge size="md">md</Badge>    // [ md ]
<Badge size="lg">lg</Badge>    // [  lg  ]
```

## Dot Indicator

When `dot` is `true`, a filled circle character is prepended inside the brackets:

```tsx
<Badge variant="success" dot>online</Badge>
// Renders: [● online]
```

## Notes

- All colors are resolved via `useTheme()`. The component never hardcodes color values.
- The `neutral` variant maps to the theme's `muted` token.
